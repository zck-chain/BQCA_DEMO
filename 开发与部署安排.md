# 📅 AI 智能网盘转换工具 —— 开发与部署安排

本部署与开发安排旨在指导前后端开发人员在本地进行依赖安装、一键启动服务、并实现工具流（上传 ➔ 预览 ➔ 校验 ➔ 一键建表并绑定）的完整功能验证。

---

## 一、 开发里程碑安排 (Roadmap & Milestones)

由于剔除了复杂的多租户空间隔离，工具的开发周期被极致压缩至 **1 人天（约 8 小时）**，各阶段任务如下：

*   **【M1：底座环境对齐】（2 小时）**：
    *   在谷歌云控制台确保 `bqca-demo` 存储桶存在，且配置好唯一的 BigQuery 外部连接。
    *   本地配置好 Python 虚拟环境与 GCP 本地开发凭证授权（Application Default Credentials）。
*   **【M2：后端服务、路由与 SQL 部署】（3 小时）**：
    *   部署两阶段 SQL 路由预览视图（在 BigQuery 中只作为动态预览层）。
    *   开发 `/api/workspace/approve` 接口：当用户确认无误时，该接口执行 DDL/DML **在 BigQuery 里动态创建或合并入持久化物理表 `t_verified_smart_drive`**，并触发 BQCA 一键静默绑定。
*   **【M3：前端大屏与一键绑定交互】（3 小时）**：
    *   实现网盘列表直接展示 GCS 桶中的临时文件。
    *   实现双屏核对弹窗，在用户微调字段后点击“确认并绑定”时，发送请求给后端、瞬间在 BigQuery 生出物理表。
    *   唤醒右下角 BQCA 对话挂件，完成完整体验。

## 二、 本地开发快速启动 (Local Setup Quickstart)

### 1. 初始化 Python 虚拟环境与依赖安装

在 `BQCA-demo` 目录下运行以下命令安装 Python 核心依赖：

```bash
# 1. 切换到项目根目录
cd /Users/apple/Desktop/工作/产品演示/BQCA-demo

# 2. 创建并激活虚拟环境
python3 -m venv venv
source venv/bin/activate

# 3. 安装依赖库
pip install fastapi uvicorn google-cloud-storage google-cloud-bigquery pydantic
```

### 2. 获取 Google Cloud 本地开发授权 (ADC)

在开发机命令行中运行以下命令进行登录，授权本地 Python 能够无缝读写 BigQuery 和 GCS：

```bash
gcloud auth application-default login
```

### 3. 一键启动后端 FastAPI 服务

在 `BQCA-demo` 目录下启动本地热重载 Uvicorn 服务器：

```bash
python3 -m backend.main
```
*   **全接口 API 调试大屏**：浏览器访问 `http://127.0.0.1:8000/docs` 即可进行所有核心 API 的独立验证。

### 4. 运行前端网盘页面

双击打开 `frontend/index.html` 即可开始直观体验。由于开启了本地跨域（CORS），网页将直接与你本地启动的 `localhost:8000` 后端进行完美的数据联调交互！

## 三、 完整功能链条验收标准 (Acceptance Criteria)

1.  **直传验证**：在前端拖入一个 PDF 文档，看到该文件成功直传到 GCS 的 `workspaces/temp_space/` 目录下。
2.  **预览验证**：点击“一键开始分析”，后端成功编译并在 BigQuery 部署两阶段路由视图，表格刷新并渲染出大模型抓取的核心元数据、置信度以及证据链。
3.  **HIL 订正验证**：点击某一行的“核对并绑定”按钮，弹出侧边板。将标题栏的错别字修改，点击保存。
4.  **建表与绑定验证**：
    *   后端接收到修改，在 BigQuery 里自动创建了物理表 `t_verified_smart_drive`，并且里面已经写入了这行刚刚修改好的高保真数据！
    *   右下角 BQCA 机器人口袋被瞬间激活，开聊回复时展现的数据，完美匹配操作员刚刚订正过的文本内容，验收完美通过！
